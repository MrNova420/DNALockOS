# Core Cryptographic Libraries
# NOTE: PyNaCl is EXCLUDED - the system falls back to cryptography library
#       If you have working build tools, you can optionally install PyNaCl:
#       pkg install libsodium && pip install PyNaCl

# ==============================================================================
# EXCLUDED FOR MOBILE (reduces build time and avoids compilation issues):
# ==============================================================================
# - PyNaCl: Requires libsodium compilation (system uses cryptography fallback)
# - uvloop: Requires libuv compilation (system uses default asyncio)
# - pyzmq: Requires libzmq compilation (ZMQ messaging disabled)
# - psycopg2-binary: Requires pg_config and PostgreSQL headers
# - asyncpg: Requires PostgreSQL development headers
# - redis: Optional - system uses in-memory fallback
# - passlib[bcrypt]: bcrypt can be slow to compile - uses fallback
# - mkdocs, mkdocs-material: Documentation tools not needed at runtime
# - black, flake8, mypy, isort: Dev tools not needed at runtime
# - pytest and testing libraries: Not needed for production
# - prometheus-client, structlog: Monitoring/logging - optional
#
# FALLBACK BEHAVIOR:
# - Crypto: cryptography library provides Ed25519 signing (secure fallback)
# - Event loop: Default asyncio (slightly slower than uvloop but works everywhere)
# - Messaging: ZMQ features disabled (no inter-service messaging)
#
# To optionally install native deps if your Termux has build tools:
#   pkg install clang make pkg-config binutils libsodium
#   pip install PyNaCl uvloop
